Ant Colony Optimization (ACO) – Live Traveling Salesman Problem (TSP) Model

Drag cities on the canvas, add or randomize points, then run ACO to find a short tour. This is a digital model to demonstrate that the ACO algorithm can be replicated and applied (here: to TSP). In an antivirus analogy, the “tour” can represent an optimized scanning/inspection order that reduces total cost/time.

What the model demonstrates

ACO uses many “ants” to build candidate tours. Each choice is biased by: (1) pheromone on an edge (learned from good tours), and (2) how close the next city is. After every iteration, pheromone evaporates and good tours deposit pheromone, so short routes become more likely.
Antivirus link (conceptual): scanning/inspection can be treated like visiting many items with a cost (time/CPU). ACO-style reinforcement can bias the system toward efficient inspection orders under changing conditions.
